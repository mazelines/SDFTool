# SDFTool
흑백 이미지로 SDF 이미지와 SDF 아틀라스를 생성하는 작은 도구입니다.

##### 기능 소개

###### SDF 생성

첫 번째 기능은 8SSEDT 알고리즘으로 SDF 이미지를 생성하고, 생성된 SDF에 보간 처리를 적용해 카툰 렌더링에 사용할 임계값 이미지를 만드는 기능입니다.

사용 예시:

- `선택 경로`를 클릭해 미리 준비한 임계값 이미지가 있는 경로를 선택합니다.

![image-20231201225115303](README.assets/image-20231201225115303.png)

![image-20231201225002686](README.assets/image-20231201225002686.png)

- `SDF 생성`을 클릭하면 현재 폴더 아래에 `output` 폴더가 생성됩니다. 생성된 SDF는 이 폴더에 저장되고, 보간으로 생성된 임계값 이미지는 하위 `SDF` 디렉터리에 저장됩니다.

![image-20231201230133056](README.assets/image-20231201230133056.png)

###### 아틀라스 생성

두 번째 기능은 같은 크기의 여러 텍스처를 하나의 아틀라스로 합치는 기능입니다.

- 사용 방식은 SDF 생성과 비슷합니다. `선택 경로`를 클릭해 텍스처가 들어 있는 경로를 선택합니다.
- 파일 이름 형식은 아래 그림처럼 `image_1_1` 형태를 권장합니다. 파일명 끝은 `_숫자_숫자` 형식으로 끝나야 합니다.

![image-20231201231743952](README.assets/image-20231201231743952.png)

- 첫 번째 행의 텍스처와 마지막 텍스처가 각각 한 장만 준비된 경우, 아래 그림처럼 `첫 행과 마지막 행에 이미지가 한 장만 있음` 옵션을 체크합니다.

![image-20231201232310521](README.assets/image-20231201232310521.png)

![image-20231201232140897](README.assets/image-20231201232140897.png)

- `아틀라스 최종 해상도`는 아틀라스를 생성한 뒤 해당 해상도로 다시 보간할 때 사용됩니다. 예를 들어 위 예시에서 SDF 텍스처 한 장의 해상도가 1024이고 9x9 아틀라스를 생성하면 결과 해상도는 9216x9216이 됩니다. 이 값이 너무 크므로 이 옵션으로 4096x4096으로 다시 보간할 수 있습니다.

##### 소스 설치

###### 프로젝트 소개

설치 절차를 이해하기 쉽도록 프로젝트 구조를 간단히 설명합니다.

이 프로젝트는 세 부분으로 구성되어 있습니다. 주요 로직은 Python으로 작성되어 있고, UI는 QML을 사용하며, SDF를 생성하는 8SSEDT 알고리즘은 C++로 작성되어 있습니다.

따라서 프로젝트를 실행하려면 Pyside6 같은 Python 의존성과 pybind11 같은 C++ 의존성을 설치해야 합니다.

###### Python 부분

Python 개발에 PyCharm을 사용한다면 `requirements.txt`를 열었을 때 가상 환경에 의존성을 자동으로 설치할 수 있습니다.

<img src="README.assets/image-20231202001225303.png" alt="image-20231202001225303" style="zoom:50%;" />

> 위의 `pyinstaller`는 exe 패키징에 사용하는 도구이므로 반드시 설치할 필요는 없습니다.

프로젝트 설정에서 가상 환경을 구성할 때 Python 인터프리터 버전은 C++ 쪽과 동일해야 합니다. 예를 들어 C++ 쪽에서 Python 3.10을 사용했다면 여기에서도 Python 3.10을 사용해야 합니다.

![image-20231202001413965](README.assets/image-20231202001413965.png)

다음으로 `main.py`의 아래 코드 몇 줄을 확인합니다.

![image-20231202002045284](README.assets/image-20231202002045284.png)

위 코드의 14번째 줄은 `SDF_Cpp.pyd`의 경로를 추가합니다. 상대 경로를 사용하므로 아래의 `import SDF_Cpp`는 오류가 나지 않아야 합니다. 하지만 15번째 줄에서는 OpenCV DLL 경로를 추가하고 있으며, 이 부분은 절대 경로입니다. 이 경로를 수정하지 않으면 아래와 같은 오류가 발생할 가능성이 큽니다.

![image-20231202002350702](README.assets/image-20231202002350702.png)

이때는 `Dependencies`라는 도구를 사용하는 것을 권장합니다.

https://github.com/lucasg/Dependencies

이 도구로 C++에서 생성한 `SDF_Cpp.pyd`를 열면 해당 pyd가 어떤 파일에 의존하는지 확인할 수 있습니다.

![image-20231202003106141](README.assets/image-20231202003106141.png)

```python
os.add_dll_directory(r"E:\Material\code\C++\OpenCV\opencv\build\x64\vc16\bin")
```

위 OpenCV 의존성 디렉터리 경로를 자신의 컴퓨터에 맞는 경로로 바꾸면 됩니다.
